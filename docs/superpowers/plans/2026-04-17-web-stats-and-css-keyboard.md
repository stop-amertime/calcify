# Web perf stats + CSS keyboard + BIOS toggle — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bring calcite-web up to parity with the CLI's perf readout, replace the JS keyboard hack with a clickable on-screen keyboard backed by existing CSS `:active` rules, and let `run-web.bat` pick between the ASM and C BIOSes.

**Architecture:** Purely additive/replacement edits to three areas:
1. `web/` — HTML status bar + speed bar + `<key-board>` DOM; worker passes `cycleCount` through tick results.
2. `../CSS-DOS/transpiler/src/template.mjs` — extend `KEYBOARD_KEYS` with arrows/Tab/Bksp; update `emitHTMLFooter`.
3. `run-web.bat` — menu-level BIOS toggle dispatching to `generate-dos.mjs` (ASM) or `generate-dos-c.mjs` (C).

No changes to calcite-core, calcite-wasm, calcite-cli, calcite-debugger, or CSS-DOS emit-css.mjs.

**Tech Stack:** Plain HTML/CSS/JS (no framework), Rust (only as reference — not edited), Node ESM (CSS-DOS transpiler), Windows cmd batch.
